# Workplace Writing Style Guide

## Tone Spectrum

| Tone | When to Use | Example |
|------|-------------|---------|
| **Formal** | C-suite comms, legal, external partners | "We are pleased to inform you..." |
| **Professional** | Cross-team updates, documentation | "Here's a summary of our Q1 progress." |
| **Conversational** | Team channels, 1:1 messages | "Quick heads up ‚Äî we're changing..." |
| **Casual** | Close teammates, social channels | "Hey! Just pushed the fix üéâ" |

**Default to professional-conversational.** Err on the side of warmth over formality.

## Email / Message Structure

### The BLUF Pattern (Bottom Line Up Front)
1. **Bottom line** ‚Äî What you need or what happened (first sentence)
2. **Background** ‚Äî Context the reader needs
3. **Details** ‚Äî Supporting information
4. **Action** ‚Äî What happens next and by when

### The SCR Pattern (Situation-Complication-Resolution)
1. **Situation** ‚Äî What's the current state?
2. **Complication** ‚Äî What's the problem or change?
3. **Resolution** ‚Äî What do you propose?

## Feedback Writing Formula

```
"When [specific situation], [what happened/impact]. 
Going forward, [suggestion/alternative]."
```

**Example:**
"When we changed the API without updating the docs (last Tuesday's deploy), three customers filed support tickets. Going forward, let's add a docs-update checkbox to the deploy checklist."

## Words to Avoid in Professional Writing

| Avoid | Use Instead |
|-------|-------------|
| "Just" (minimizer) | Remove it or be direct |
| "Obviously" / "Clearly" | Remove ‚Äî if it were obvious, you wouldn't need to say it |
| "ASAP" | Give a specific deadline |
| "Per my last email" | "As mentioned in [link]" or just restate |
| "Going forward" (overused) | "From now on" / "Starting [date]" / just remove |
| "Synergy" / "Leverage" | Use plain English |
